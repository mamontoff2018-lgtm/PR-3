### Ответы на контрольные вопросы

1. **Что такое ViewBinding и в чём его преимущество перед классическим методом findViewById()?**  
   ViewBinding — это технология, которая автоматически генерирует класс с прямыми ссылками на все View из XML.  
   Преимущества: безопасный доступ к элементам (без `null`), нет `findViewById`, быстрее и надёжнее.  
   Подключение: в `build.gradle` (Module) добавить `viewBinding { enabled = true }` и пересинхронизировать проект.

2. **В чём разница между декларативной (XML-атрибут onClick) и программной (setOnClickListener) подпиской на событие?**  
   Декларативный способ — проще, но менее гибкий (метод должен быть public и в Activity).  
   Программный способ (`setOnClickListener`) — более современный, гибкий, позволяет динамически менять поведение.  
   Рекомендуется использовать программный подход.

3. **Что произойдёт, если в методе-обработчике, указанном в XML, изменить сигнатуру?**  
   Приложение упадёт с ошибкой `java.lang.IllegalStateException: Could not find method...`.  
   Android ищет метод строго по имени и сигнатуре `public void имяМетода(View view)`.

4. **Опишите жизненный цикл Activity. В каком методе лучше всего инициализировать слушатели для кнопок?**  
   Основные методы: `onCreate` → `onStart` → `onResume` → `onPause` → `onStop` → `onDestroy`.  
   Слушатели лучше всего инициализировать в `onCreate` — один раз при создании Activity.

5. **Что такое анонимный внутренний класс? Как он используется при установке слушателей?**  
   Анонимный внутренний класс — класс без имени, создаётся прямо в месте использования.  
   Пример: `button.setOnClickListener { ... }` или `new View.OnClickListener() { public void onClick... }`.
